## Context

Mid-wayne-plan, the user noticed layered instruction-file design (opencode-style
AGENTS.md discover-and-merge) was never discussed. Branched back to mind-explode. User's
framing: "opencode 化的 Hermes agent" — wants Hermes's clean three-tier prompt separation
but with opencode's directory-walk + merge for project instructions. Resolves the user's
own stated drift worry (persona vs AGENTS.md) by keeping three independent concern-owners.

## Research Inputs

- **Hermes prompt assembly** (hermes-agent.nousresearch.com/docs/developer-guide/prompt-assembly):
  3 cached tiers — Stable (`SOUL.md`, 20k cap), Context (project context files discovered
  CWD→git-root, **first-match-wins, only one loads**: `.hermes.md`/`HERMES.md` > `AGENTS.md`
  > `CLAUDE.md` > `.cursorrules`; 20k cap 70/20 head/tail), Volatile (`MEMORY.md`+`USER.md`+
  external memory). All frozen at session start for cache.
- **opencode rules** (opencode.ai/docs/rules): local walk-up from CWD (`AGENTS.md` > `CLAUDE.md`)
  + global `~/.config/opencode/AGENTS.md` + `~/.claude/CLAUDE.md`; **combined/concatenated, not
  overridden**; first-filename-wins per category; `opencode.json` `instructions` adds globs.
- **KB lesson `persona-shared-lift-and-first-time-tracking`**: Wayne control-plane already
  runs L1/L2/L3 layered instructions (shared SoT + cite-compression ceiling, ~4 lines/cite,
  ~50 LOC shared-file ceiling). Same mental model — Alfred's instruction layering is isomorphic.
- **KB lesson `ssot-drift-and-schema-migration`**: same conceptual state in 2 places drifts;
  keep one writer/owner per concept.
- **KB lesson `llm-prompt-and-boundary-contracts`**: instruction text is high-attention
  finite channel; keep machine-binding invariants small; prompt+parser deploy-coupled.
